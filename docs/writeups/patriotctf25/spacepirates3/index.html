ABOUT WRITEUPS MEMBERS ACHIEVEMENTS

Writeup by bllry

Space Pirates 3

Description

You are provided with a go binary.

Walkthrough

The binary is checking for a 30-byte input that, after six reversible transforms, equals the 30-byte TARGET. Because every step is a bijection (XOR, rotations, swaps, modular add/sub, chunk reverse), you can invert them in reverse order on TARGET to recover the required plaintext input.

What the program does (forward)

processVault() takes a 30-byte input and applies, in order:

  • XOR with repeating 7-byte key
  • Rotate-left each byte by a repeating 8-byte rotation pattern
  • Swap adjacent byte pairs
  • Subtract constant 0x93 from every byte
  • Reverse each 6-byte chunk
  • XOR each byte with (i^2 + i) mod 256

Then it checks whether the result equals TARGET. So we need an input such that:

op6(op5(op4(op3(op2(op1(input)))))) == TARGET

Inverting the operations (reverse order)

All 6 ops are bijective, so invert them in reverse order:

  • Inv6: XOR with same (i^2 + i) (XOR is self-inverse).
  • Inv5: Reverse chunks of 6 again (reversal is self-inverse).
  • Inv4: Add 0x93 to each byte (undo subtraction).
  • Inv3: Swap adjacent pairs again (self-inverse).
  • Inv2: Rotate right by the same pattern (undo rotate-left).
  • Inv1: XOR with the same 7-byte key again.

Recovered input / flag

Applying those inverses to the given TARGET yields the ASCII bytes:

PCTF{M4ST3R_0F_TH3_S3V3N_S34S}

@ DEAD PIRATE SOCIETY
